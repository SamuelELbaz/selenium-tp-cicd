1. Les tests sont plus rapides à exécuter en automatisation qu'en manuel, cela permet également de clairement lister les différents tests qu'on veut faire et d'avoir un rapport des tests qu'on fait.
   Le CI/CD améliore la qualité du code en automatisant les tests et les déploiements, cela permet de détecter rapidement les erreurs et de les corriger avant qu'elles n'affectent les utilisateurs finaux.

2. J'ai rencontré des difficultés dans tous les aspects de Selenium, que ce soit pour l'installation ou la configuration.
   Je pourrais améliorer la stabilité des tests en testant sur différents navigateurs, et en envoyant une notification en cas d'échec.

3. Les métriques les plus importantes pour moi sont la réussite des tests, leur temps d'execution.
